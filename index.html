The cascade:

  1. Select element; filter rules
  2. Cascade order W3C, MDN
    1. User agent
    2. User agent !important
    3. User
    4. Author
    5. CSS Animations (doesn't cascade)
    6. Author !important
    7. User !important
  3. Specificity

BEM vs Atomic CSS

  • Atomic CSS gets a bad rep from old school web devs because it's *weird*
  • Give it a minute, and you'll find it's actually pretty awesome
  • Problem is styling info in markup
  • Also, lots of repetition as you call multiple classes
  • We're gonna start by looking at two differing schools of thought when it comes to naming selectors
  • BEM stands for Block Element Modifier
  • It's a namespacing convention to help isolate your CSS
  • One class per component
  • Atomic CSS is the idea that classes should be reused

DRY - but HTML and CSS are fundamentally linked.

Consider a situation where you have no control over the markup. And the markup is bad. What is your CSS going to look like?

Bad.

We need a language robust enough to not force us in a corner of specificity hacks when we can't control the markup

The descendent selector is the most expensiveMDN.
  • Browsers parse CSS from right to left
  • When it comes to writing efficient selectors, specific things like IDs are ideal
  • General selectors like element-level ones are very inefficient

"best practice" and efficiency

Functions!

Here's where things get interesting.

Mixins in SASS

              
                @mixin dark-background($color){
                  background-color: $color;
                  color: white;
                  text-shadow: 0 0 0.5rem black;
                  font-weight: 700;
                }

                .box {
                  @include dark-background(black);
                }

                .different-box {
                  @include dark-background(blue);
                }

Mixins in your precompiler means less repetition in development, but they become so much more powerful in raw CSS.

Remember the BEM vs Atomic problem?

  • BEM is inelegant, clunky and repetitive
  • Atomic means holding styling information in semantic markup - bad for separation of duties

But what happens when we put it in raw CSS?

              
                @function button($color) {
                  padding: 1em;
                  margin: 0 1em;
                  background-color: $color;
                }

                @function display-font {
                  font-family: 'Playfair Display';
                  letterspacing: -0.01em;
                  line-height: 1em;
                }

                .block--element {
                  function: button(blue);
                  function: display-font;
                }

Come at me, efficiency !

              
                @function button($color) {
                  padding: 1em;
                  margin: 0 1em;
                  background-color: $color;
                }

                @function display-font {

                  letterspacing: -0.01em;
                  line-height: 1em;
                }

                #this-specific-element {
                  function: button(blue);
                  function: display-font;
                }

No styling in markup? Check.

Ease of prototyping? Check.

Separation of concerns

Inherit from parent

  • Settings
  • Typography
  • Aural

Don't inherit

  • Layout
  • Decoration

Explicity separating Inherit, Layout and Decoration

              
                #foo {
                  &inherit {
                    function: article-font;
                  }
                  &layout {
                    display: block;
                    position: absolute;
                    function: container(320px);
                  }
                  &decoration {
                    color: pink;
                    function: random-background;
                  }
                }

Separation of concerns

  • Separate properties that propagate down
  • Visually clear
  • Makes theming easier
  • A note on performance: reflows are much more expensive than repaints. This could make performance debugging clearer too.
  • Async loading

Taking on Specificity Wars

  • Protection from rogue inheritence with all: initial
  • Bonus: Seperation of concerns
